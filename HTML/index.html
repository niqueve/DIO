Entendendo Comunicação Client x Server


História da Web | voltar

A Web, como conhecemos nos dias de hoje, foi criada pelo inglês Tim Berners-Lee num projeto apresentado ao mundo em 1989, porém sua história remonta desde o surgimento da primeira geração de computadores.

  • Primeira geração
    • O Harverd Mark I (1944) | Colossus(1946) | ENIAC(1946)
  • Segunda geração
    • Transistores(1959 - 1965)
  • Terceira geração
    • Circuito Integrado (1965 - 1970)
  • Quarta geração
    • Computadores pessoais (a partir de 1971)
  • Quinta geração
    • IA?

*Algumas pessoas divergem sobre a IA ser ou não o marco que nos fará entrar na quinta geração, porém não é possivel negar toda a sua evolução e revolução causada.


As telecomunicações foram os antecessores da internet e sua estrutura facilitou o processo. O telegrafo foi o primeiro representante moderno (1958).
A ideia para a web surgiu e se desenvolveu na decada de 1950 durante a Guerra Fria. A DARPA com a ARPANET(1969) foi a primeira a implementar o protocolo de rede NCP e de internet TCP/IP, sendo o ponto inicial e a base técnica da rede de computadores mundial, porém seu acesso era restrito para fins de transmissão de dados militares sigilosos e pesquisa nos Estados Unidos. A ARPANET seguiu ativa até 1990.

Até a criação da ARPANET as comunicações eram feitas ponta a ponta. Com o seu surgimento surgiu a comutação de pacotes(uma única ligação para comunicação entre vários pares).

Clients | voltar

Os "Clients" nada mais são do que os usuários (notebooks, computadores, celulares..) que enviam uma solicitação através da internet (protocolo HTTP) para o server (servidor que armazenas os dados). São retornados arquivos estáticos a solicitação (html, css, javaScript, imagens..) e a velocidade de resposta varia de acordo com a velocidade de conexão.

Servers | voltar

Os Servers (servidores) armazenam ou fazem a ponte entre os clientes e os dados solicitados. Quando acessamos um banco de dados, a solicitação é feita ao servidor e o sevidor faz o acesso retornando o que foi solicitado. Ele recebe uma solicitação HTTP e retorna aquivos estáticos.

Estrutura básica do html | voltar


            <!DOCTYPE html>
            <html>
            <body>
            
            <h1>Meu primeiro título;/h1>
            <p>Meu primeiro parágrafo.</p>
            
            </body>
            </html>

A estrutura básica de todo código html (código acima),começa com uma declaração DOCTYPE seguida pela tag < html > que marca e indica ao navegador como o documento deve ser tratado.
Dentro da tag html, temos a tag <body> que separa tudo que será exibido ao usuário. Já a tag <p> diz ao navegador que o texto inserido entre as tags é um novo parágrafo.

Tags e seus atributos | voltar

Tags nada mais são do que marcadores para que o navegador saiba como lidar com o que está escrito. Toda tag (apenas com algumas excessões) devem ser abertas <p> e fechadas </p>.
Alguns exemplos de tags são: <p>, <h1> e <body>. As tags podem ser estruturais, semânticas, para formatação de textos, tabelas, adicionar links e formulários.

Listas | voltar

Há 2 tipos de listas no HTML, as ordenadas <ol> e as não ordenadas <ul> 


    <ul>
        <li>Item de lista não ordenada 1</li>
        <li>Item de lista não ordenada 2</li>
        <li>Item de lista não ordenada 3</li>
    </ul>
    
    <ol>
        <li>Item de lista ordenada 1</li>
        <li>Item de lista ordenada 2</li>
        <li>Item de lista ordenada 3</li>
    </ol>
    
    <dl>
        <dt>Termo de definição 1</dt>
        <dd>Descrição do termo 1</dd>
        <dt>Termo de definição 2</dt>
        <dd>Descrição do termo 2</dd>
    </dl>

        <a href="mailto:exemplo@exemplo.com">Enviar um e-mail</a>
